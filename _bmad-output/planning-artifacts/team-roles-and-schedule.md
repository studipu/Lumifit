# 팀 운영 문서

## 팀 구성 및 역할 분담

| 이름/역할 | 담당 영역 | 주요 책임 | 핵심 산출물 |
|---|---|---|---|
| 팀장 / PM / 통합 리드 | 기획, 통합, 품질 관리 | PRD/아키텍처 관리, 범위 조정, API/데이터 구조 승인, 전체 QA, 발표 총괄 | PRD, 아키텍처, 통합 시나리오, 발표 자료 |
| Frontend Lead | 사용자 웹 | 랜딩, 체형 입력, 2D 아바타 선택, 쇼룸/상품 상세, 3D 상태/뷰어 UI 구현 | 사용자 웹 화면, 프론트 UI 흐름 |
| Backend & Data Lead | API, DB, 비즈니스 로직 | DB 스키마 설계, 체형 프로필 API, 상품 API, 생성 요청/조회/재사용 로직 구현 | DB 스키마, API 서버, 데이터 모델 |
| DevOps & Pipeline Lead | 3D 생성 파이프라인, 배포, 운영 기능 | VARCO 연동, worker, queue, object storage, Kubernetes, 관리자 기능 일부 구현 | 생성 파이프라인, 배포 환경, 운영 기능 |

## 공통 협업 규칙

| 항목 | 내용 |
|---|---|
| 공통 데이터 구조 | 체형 프로필, 2D 아바타 응답, 3D 생성 응답 구조를 먼저 고정 |
| 통합 기준 | `상품-아바타-옵션` 조합 키를 공통 규칙으로 사용 |
| 커뮤니케이션 | API 변경은 즉시 공유, 주 1회 통합 점검 |
| 우선순위 | 사용자 핵심 흐름인 `체형 입력 -> 아바타 선택 -> 상품 상세 -> 3D 생성`을 최우선 구현 |
| 발표 준비 | 6주차부터 시연 흐름과 발표 역할을 병행 정리 |
| QA 방식 | 매주 말 실제 실행 가능한 데모 기준으로 점검 |

## 발표용 역할 소개 문구

- 팀장 / PM / 통합 리드: 제품 기획, 시스템 구조 설계, 전체 통합 및 발표 총괄 담당
- Frontend Lead: 일반 사용자가 직접 사용하는 웹 화면과 사용자 경험 구현 담당
- Backend & Data Lead: 서버 API, 데이터 구조, 프로필/상품/3D 재사용 로직 구현 담당
- DevOps & Pipeline Lead: 3D 생성 파이프라인, 배포 환경, 운영 기능 및 인프라 담당
